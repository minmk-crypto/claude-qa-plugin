# QA 우선순위 기준

서비스 오픈 시 **유저에게 미치는 영향** 기준으로 분류한다.

| 등급 | 유저 영향 | 판단 기준 | 예시 |
|------|----------|----------|------|
| **매우높음** | 서비스 이용 불가 또는 치명적 혼란 | 핵심 기능이 완전히 깨지거나, 유저가 잘못된 정보로 의사결정하게 되는 결함 | 구매/결제 불가, 가격 미노출/오표시, 페이지 로드 실패(404/빈 화면), 장바구니 고스트 상태, 핵심 CTA 미동작 |
| **높음** | 기본 유즈케이스가 깨짐 | 주요 사용 시나리오를 완수할 수 없거나 심각한 불편을 유발하는 결함 | 필터/정렬 미동작, 검색 불가, 폼 검증 누락, 상태 전환 불일치(>500ms), 정책 불일치(노출 조건 위반), 자동탐색 기능 요소 미동작 |
| **보통** | 불편하지만 기능은 동작 | 유저가 목적을 달성할 수는 있으나 경험이 저하되는 결함 | 마진/폰트/간격 수치 차이, 반응형 레이아웃 깨짐, 이미지 비율 미세 차이, 로딩 인디케이터 미노출, 네비게이션 불일치, 전환 품질 경고(100~500ms) |
| **낮음** | 사소한 차이 | 대부분의 유저가 인지하지 못하거나 기능에 영향이 없는 결함 | 호버 효과 차이, 애니메이션 미적용, 접근성 미흡, 콘솔 경고, 폰트 렌더링 미세 차이 |

### 판단 원칙

1. **"유저가 돈을 쓰거나 핵심 의사결정을 하는 흐름"이 깨지면 매우높음**
2. **"유저가 원하는 걸 찾고 비교하는 흐름"이 깨지면 높음**
3. **"기능은 되지만 불편하거나 어색하면" 보통**
4. **"대부분 유저가 모르고 지나가면" 낮음**
