# QA 우선순위 기준

TC 생성 시 자동 분류에 사용.

## 등급 정의

| 등급 | 정의 | 예시 |
|------|------|------|
| **매우 높음** | 핵심 기능 불가 | 구매 불가, 탐색 차단, 필수 정보 미노출 |
| **높음** | 핵심 기능에 불편함 or 운영 지장 | 버튼 미동작, 정책 불일치, 필터/정렬 미동작 |
| **보통** | 어색함 (불편하진 않음) | 디자인 수치 차이, 미세 정렬, 반응형 전환 |
| **낮음** | 그 외 | 호버 효과, 애니메이션, 접근성 |

## 자동 분류 규칙

### 매우 높음
- 구매/결제 관련 요소 (ATC, checkout, payment, cart)
- 가격 표시 없음 / NaN / 빈값
- 상품명/이미지 미노출
- 404 링크
- 페이지 로드 실패

### 높음
- 버튼 클릭해도 동작 없음
- 폼 제출 에러
- Figma/PRD 정책과 실제 불일치
- 필터/정렬 UI는 있으나 동작 안 함
- 품절 처리 미흡

### 보통
- 마진/패딩 수치 차이
- 폰트 크기/굵기 차이
- 이미지 비율 미세 차이
- 해상도별 레이아웃 전환

### 낮음
- 호버/포커스 효과
- 애니메이션/트랜지션
- aria-label 누락
- 콘솔 경고
